# FishMindOS Agent 文档体系

本目录包含定义FishMindOS Agent（智能机器狗大脑）的所有文档。

## 📚 文档列表

### 1. identity.md - 身份定义
**定义**: Agent的自我认知和存在意义

**包含内容**:
- 物理形态描述
- 能力范围
- 语言风格（第一人称使用规则）
- 边界认知（知道什么、不知道什么）
- 核心价值

**用途**: 让LLM理解"我是谁"

---

### 2. agent.md - Agent定义
**定义**: Agent的角色、使命和行为准则

**包含内容**:
- 身份认同（名字、形态、自我指称）
- 核心使命（理解、执行、感知、反馈）
- 思维方式（思考流程、决策原则）
- 语言风格（应该做的、不应该做的）
- 安全准则
- 个性特征

**用途**: 让LLM理解"我应该怎么做"

---

### 3. tools.md - 工具定义
**定义**: 所有可用技能的详细说明

**包含内容**:
- 每个技能的名称、功能、参数
- 参数类型、是否必需、示例值
- 使用场景和示例
- 复杂任务的完整流程示例
- 工具调用流程

**用途**: 让LLM知道"我能做什么"和"怎么做"

---

## 🎯 使用方式

### 作为系统提示词

将这些文档整合为LLM的系统提示词：

```markdown
# 系统提示词

## 身份 (from identity.md)
[identity.md 的内容]

## 角色 (from agent.md)
[agent.md 的内容]

## 工具 (from tools.md)
[tools.md 的内容]

## 当前状态
- 位置: {current_location}
- 电量: {battery_level}%
- 携带物品: {carrying_item}
- 当前地图: {current_map}
```

### 动态加载

在实际应用中，可以：
1. 启动时加载所有文档
2. 整合为系统提示词
3. 每次对话时附加当前状态
4. 发送给LLM进行处理

### 文档更新

当新增技能或修改行为时：
1. 更新相应的.md文件
2. 重启系统或重新加载提示词
3. LLM会立即使用新的定义

## 📝 文档编写规范

### identity.md 编写要点
- 强调物理存在（不是AI，是机器狗）
- 明确能力边界
- 定义语言风格规则
- 列出核心价值和态度

### agent.md 编写要点
- 清晰的使命描述
- 具体的思考流程
- 明确的行为准则（应该/不应该）
- 安全边界

### tools.md 编写要点
- 每个技能的完整描述
- 参数的类型和含义
- 使用示例
- 复杂任务的组合示例

## 🎨 设计原则

1. **清晰性**: 每个文档都有明确的主题和边界
2. **完整性**: 覆盖Agent需要知道的所有信息
3. **一致性**: 文档之间没有矛盾
4. **可扩展性**: 容易添加新内容
5. **LLM友好**: 结构清晰，方便LLM理解和使用

## 🚀 快速开始

```python
# 加载Agent定义文档
with open("docs/identity.md", "r") as f:
    identity = f.read()

with open("docs/agent.md", "r") as f:
    agent = f.read()

with open("docs/tools.md", "r") as f:
    tools = f.read()

# 整合为系统提示词
system_prompt = f"""
# 系统提示词

## 身份
{identity}

## 角色
{agent}

## 工具
{tools}
"""

# 发送给LLM
response = llm.chat(
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": "去会议室"}
    ]
)
```

## 📖 阅读顺序建议

1. **先读 identity.md** - 理解"我是谁"
2. **再读 agent.md** - 理解"我应该怎么做"
3. **最后读 tools.md** - 理解"我能做什么"

这样能帮助LLM建立完整的自我认知。
